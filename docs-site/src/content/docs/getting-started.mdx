By the end of this page, you'll have all artifacts installed and running your first guided workflow.

<Steps>

1. **Prerequisites**

   <Tabs syncKey="platform">
     <TabItem label="VS Code">
       - [VS Code](https://code.visualstudio.com/download) 1.113.0+ with the [GitHub Copilot Chat](https://marketplace.visualstudio.com/items?itemName=GitHub.copilot-chat) extension
       - GitHub Copilot active subscription, signed in via the Copilot extension
       - Node.js 18+ (required for MCP servers)
       - Enable nested sub-agent workflows:

         <LinkButton href="vscode://settings/chat.subagents.allowInvocationsFromSubagents" variant="secondary" icon="setting" iconPlacement="start" class="sl-text-xs" style="padding: 0.25rem 0.75rem;">chat.subagents.allowInvocationsFromSubagents</LinkButton>
     </TabItem>
     <TabItem label="GitHub Copilot CLI">
       - [GitHub Copilot CLI](https://github.com/features/copilot/cli) 1.0.6+
       - GitHub Copilot active subscription, authenticated via `gh auth login`
       - Node.js 18+ (required for MCP servers)
     </TabItem>
   </Tabs>

2. **Installation**

   <Tabs syncKey="platform">
     <TabItem label="VS Code">
       <LinkButton href="vscode://chat-plugin/install?source=microsoft/devsquad-copilot" variant="primary" icon="rocket" iconPlacement="start" class="sl-text-sm" style="padding: 0.35rem 1rem;">Install</LinkButton>

       If you see an error, check the [Troubleshooting](/troubleshooting/) page.
     </TabItem>
     <TabItem label="GitHub Copilot CLI">
       ```bash
       copilot plugin install microsoft/devsquad-copilot
       ```

       <Aside type="tip" title="Unattended Mode">
         The `--yolo` flag enables unattended mode, where the agent operates autonomously by automatically approving terminal commands, file changes, and tool usage without requiring user confirmation.

         ```bash
         copilot --agent devsquad:devsquad --yolo
         ```
       </Aside>

       If you see an error, check the [Troubleshooting](/troubleshooting/) page.
     </TabItem>
   </Tabs>

3. **Select an Agent**

   <Tabs syncKey="platform">
     <TabItem label="VS Code">
       - **Guided** (recommended for new users): open the Chat panel and type `/agents devsquad`. The conductor walks through each phase in sequence, asking Socratic questions and delegating to the right specialist.

         ```
         /agents devsquad
         ```

       - **Direct**: select a specific agent for targeted work.

         ```
         /agents devsquad.implement    # Go directly to implementation
         /agents devsquad.specify      # Go directly to specification
         /agents devsquad.security     # Run a security assessment
         ```

       Both modes use the same agents, skills, and quality gates. Guided mode adds orchestration and phase transitions.
     </TabItem>
     <TabItem label="GitHub Copilot CLI">
       - **Guided** (recommended for new users): use the conductor as the entry point. It walks through each phase in sequence, asking Socratic questions and delegating to the right specialist.

         ```bash
         copilot --agent devsquad:devsquad
         ```

       - **Direct**: invoke a specific agent for targeted work using `devsquad:<agent>`.

         ```bash
         copilot --agent devsquad:devsquad.implement    # Go directly to implementation
         copilot --agent devsquad:devsquad.specify      # Go directly to specification
         copilot --agent devsquad:devsquad.security     # Run a security assessment
         ```

       Both modes use the same agents, skills, and quality gates. Guided mode adds orchestration and phase transitions.
     </TabItem>
   </Tabs>

4. **Project Initialization**

   <Tabs syncKey="platform">
     <TabItem label="VS Code">
       If you selected the conductor (`devsquad`), it will detect a new project and guide you through initialization automatically.

       Alternatively, select `devsquad.init` directly to scaffold your project.
     </TabItem>
     <TabItem label="GitHub Copilot CLI">
       If you selected the conductor, it will detect a new project and guide you through initialization automatically.

       Alternatively, run the init agent directly:

       ```bash
       copilot --agent devsquad:devsquad.init
       ```
     </TabItem>
   </Tabs>

   The init agent creates:

   - **Config files**: `copilot-instructions.md`, instruction files for specs/ADRs/tasks, coding guidelines, markdownlint config.
   - **Documentation templates**: Templates for features, migrations, envisioning, and ADRs.
   - **Community files** (optional): `SECURITY.md`, `CONTRIBUTING.md`, `LICENSE`, `CODE_OF_CONDUCT.md`.

   After initialization, your project structure will look like this:

   <FileTree>
   - .github/
     - docs/
       - coding-guidelines.md
     - instructions/
       - adrs.instructions.md
       - documentation-style.instructions.md
       - envisioning.instructions.md
       - migration-specs.instructions.md
       - migration-tasks.instructions.md
       - specs.instructions.md
       - tasks.instructions.md
     - copilot-instructions.md
   - docs/
     - architecture/
       - decisions/
         - ADR-TEMPLATE.md
     - envisioning/
       - TEMPLATE.md
     - features/
       - TEMPLATE.md
     - migrations/
       - TEMPLATE.md
   - .markdownlint.yaml
   </FileTree>

5. **Your First Workflow**

   <Tabs syncKey="platform">
     <TabItem label="VS Code">
       Open the Chat panel and select agents via `/agents`:

       1. **Envision** (`/agents devsquad.envision`): Capture the strategic vision through structured questions about customer, pain points, and goals.
       2. **Kickoff** (`/agents devsquad.kickoff`): Structure the project hierarchy with epics and features. Sync with GitHub Issues, Azure DevOps, or store locally in tasks.md.
       3. **Specify** (`/agents devsquad.specify`): Write feature specifications with user stories, acceptance criteria, and compliance requirements.
       4. **Plan** (`/agents devsquad.plan`): Generate technical design artifacts. Create Architecture Decision Records for significant choices.
       5. **Decompose** (`/agents devsquad.decompose`): Break specifications into user stories and tasks. Create work items on GitHub Issues, Azure DevOps, or store locally in tasks.md.
       6. **Implement** (`/agents devsquad.implement`): Execute implementation from a task, issue, or work item. Quality gates and branch management included.
       7. **Review** (`/agents devsquad.review`): Validate the implementation against specs, ADRs, and the plan with independent context.
     </TabItem>
     <TabItem label="GitHub Copilot CLI">
       Use agents via the `--agent` flag:

       1. **Envision** (`devsquad:devsquad.envision`): Capture the strategic vision through structured questions about customer, pain points, and goals.
       2. **Kickoff** (`devsquad:devsquad.kickoff`): Structure the project hierarchy with epics and features. Sync with GitHub Issues, Azure DevOps, or store locally in tasks.md.
       3. **Specify** (`devsquad:devsquad.specify`): Write feature specifications with user stories, acceptance criteria, and compliance requirements.
       4. **Plan** (`devsquad:devsquad.plan`): Generate technical design artifacts. Create Architecture Decision Records for significant choices.
       5. **Decompose** (`devsquad:devsquad.decompose`): Break specifications into user stories and tasks. Create work items on GitHub Issues, Azure DevOps, or store locally in tasks.md.
       6. **Implement** (`devsquad:devsquad.implement`): Execute implementation from a task, issue, or work item. Quality gates and branch management included.
       7. **Review** (`devsquad:devsquad.review`): Validate the implementation against specs, ADRs, and the plan with independent context.
     </TabItem>
   </Tabs>

</Steps>
