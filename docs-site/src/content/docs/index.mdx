## From Intent to Implementation

Delivering at the pace of innovation means structuring delivery so that speed and quality reinforce each other. The framework organizes work in three layers — each produces artifacts that reduce rework downstream:

<CardGrid>
  <Card title="Intent — Why" icon="rocket">
    Capture the strategic motivation through structured envisioning. Questions surface customer pain points, goals, and business context before any code is written.
  </Card>
  <Card title="Specification — What" icon="document">
    Feature and migration specs define scope, acceptance criteria, and constraints. Written for stakeholders, these artifacts become the contract that downstream work is measured against.
  </Card>
  <Card title="Architecture Decisions — How" icon="setting">
    ADRs record how and why specific technical approaches were selected. Evaluated against ranked priorities, not generic pros/cons — so decisions stay traceable.
  </Card>
</CardGrid>

## Intentional AI

Across every layer, agents operate with intent — not generating code blindly, but structuring thought at every step.

<CardGrid>
  <Card title="Ask Before Assuming" icon="approve-check">
    Agents verify understanding and surface trade-offs before acting. Clarifying questions replace guesswork.
  </Card>
  <Card title="Traceable Decisions" icon="document">
    Every choice is recorded with reasoning, alternatives considered, and confidence level.
  </Card>
  <Card title="Human in Control" icon="warning">
    No medium or high impact change executes without explicit approval. The developer decides; the AI accelerates.
  </Card>
  <Card title="Comprehension Over Speed" icon="open-book">
    Post-implementation checks verify the developer understands what was generated and why.
  </Card>
</CardGrid>

## Delivery Flow

<Steps>
1. **Envision** - Capture strategic vision and business context
2. **Kickoff** - Structure project hierarchy and sync with the board
3. **Specify** - Produce detailed feature or migration specifications
4. **Plan** - Generate technical design artifacts and architecture decisions
5. **Decompose** - Break specifications into user stories and tasks
6. **Implement** - Execute implementation with guardrails and quality gates
7. **Review** - Validate implementation against specs, ADRs, and the plan
</Steps>

Each stage can be revisited as new information emerges. The framework supports iterative refinement without losing traceability.
