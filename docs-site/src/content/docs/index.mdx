## What is DevSquad?

DevSquad implements **Intent-Driven Development (IDD)**, a methodology built on three pillars:

<CardGrid>
  <Card title="Intent (Why)" icon="rocket">
    The strategic motivation behind the project, captured through structured envisioning with Socratic questions about customer, pain points, and goals.
  </Card>
  <Card title="Specification (What)" icon="document">
    Precise feature and migration specs that define scope, acceptance criteria, and constraints. Written for stakeholders, not developers.
  </Card>
  <Card title="Architecture Decisions (How)" icon="setting">
    ADRs that record how and why specific technical approaches were selected. Evaluated against ranked priorities, not generic pros/cons.
  </Card>
  <Card title="Socratic AI" icon="approve-check">
    Agents ask clarifying questions, surface trade-offs, and present options. Every workflow is human-in-the-loop by design.
  </Card>
</CardGrid>

## Delivery Flow

<Steps>
1. **Envision** - Capture strategic vision and business context
2. **Kickoff** - Structure project hierarchy and sync with the board
3. **Specify** - Produce detailed feature or migration specifications
4. **Plan** - Generate technical design artifacts and architecture decisions
5. **Decompose** - Break specifications into user stories and tasks
6. **Implement** - Execute implementation with guardrails and quality gates
7. **Review** - Validate implementation against specs, ADRs, and the plan
</Steps>

Each stage can be revisited as new information emerges. The framework supports iterative refinement without losing traceability.
